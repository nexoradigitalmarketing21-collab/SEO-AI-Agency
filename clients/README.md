# Client Workspace Structure

## Purpose

This document defines the standardized folder structure for each client project. Every new client gets the same organized workspace to ensure consistency, efficiency, and easy navigation.

**Benefits:**
- Consistent organization across all clients
- Easy to find files and deliverables
- Clear workflow progression
- Scalable structure
- Professional presentation

---

## Standard Client Folder Structure

```
clients/
└── [Client Name]/
    ├── 01 Discovery/
    ├── 02 Audit/
    ├── 03 Keyword Research/
    ├── 04 Strategy/
    ├── 05 Content/
    ├── 06 Reports/
    ├── 07 Deliverables/
    └── 08 Assets/
```

---

## Folder Descriptions

### 01 Discovery/

**Purpose:** Store all initial client information, discovery call notes, and onboarding materials.

**Files:**
- `client-brief.md` - Client information, business details, goals
- `discovery-call-notes.md` - Notes from discovery call
- `questionnaire-responses.md` - Client questionnaire answers
- `access-credentials.md` - Login credentials (secure)
- `contract-signed.pdf` - Signed agreement
- `onboarding-checklist.md` - Onboarding progress tracker

**Subfolders:**
- `proposals/` - All proposal versions
- `contracts/` - Contracts and agreements
- `invoices/` - Invoices and payments

## File Naming Conventions

### Standard Format

```
[date]-[type]-[description].[extension]

Examples:
2026-01-15-audit-technical-seo.md
2026-01-20-report-monthly-january.pdf
2026-02-01-brief-blog-post-seo-tips.md
```

### Date Format
- YYYY-MM-DD (2026-01-15)

### Types
- `audit` - Audit reports
- `report` - Performance reports
- `brief` - Content briefs
- `strategy` - Strategy documents
- `content` - Content pieces
- `presentation` - Presentations

### Descriptions
- Be specific and clear
- Use hyphens for spaces
- Keep it concise but descriptive

## Access and Security

### Sensitive Information

**Files requiring extra security:**
- `access-credentials.md` - Login credentials
- `contracts/` - Legal documents
- `invoices/` - Financial information

**Security Measures:**
- Store in encrypted folder
- Limit access to authorized personnel
- Use password manager
- Regular backups

### Client Access

**What clients can access:**
- Deliverables folder
- Reports folder
- Approved content

**What clients cannot access:**
- Internal working files
- Credentials
- Financial information
