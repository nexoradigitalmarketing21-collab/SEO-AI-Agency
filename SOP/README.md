# SOP - Standard Operating Procedures

## Purpose

The SOP/ directory contains **Standard Operating Procedures** for the Nexora AI SEO Agency. These are detailed, step-by-step instructions for recurring tasks and processes that ensure consistency, quality, and efficiency across all agency operations.

**SOPs bridge the gap between high-level workflow rules (brain/workflow-rules.md) and day-to-day agent execution.**

---

## How SOPs Work

### For Agents

Agents **must follow** relevant SOPs when:
- Executing standard client services
- Performing repetitive tasks
- Following established procedures
- Ensuring quality and consistency

**Example:**
```
Task: Conduct SEO Audit
1. Consult brain/workflow-rules.md (understand workflow)
2. Read SOP/audit-process.md (detailed steps)
3. Follow step-by-step procedure
4. Use deliverables/seo-audit/template.md
5. Validate against brain/quality-standards.md
```

### For Humans

Humans use SOPs to:
- Train new team members
- Ensure process consistency
- Onboard new clients
- Maintain quality standards
- Scale operations

## SOP Structure

### Standard SOP Format

Every SOP includes:

**1. Purpose & Scope**
- What the SOP covers
- When to use it
- Who should follow it

**2. Prerequisites**
- Required tools and access
- Necessary information
- Prior approvals needed

**3. Step-by-Step Procedure**
- Numbered steps
- Detailed instructions
- Decision points
- Quality checks

**4. Templates & Forms**
- Required templates
- Checkboxes and forms
- Example outputs

**5. Quality Standards**
- Minimum requirements
- Review process
- Approval criteria

**6. Troubleshooting**
- Common issues
- Solutions
- Escalation paths

**7. Related Documents**
- Linked SOPs
- Referenced templates
- Supporting materials

## Using SOPs

### For Standard Tasks

**When to Use SOPs:**
- Recurring client services
- Standard procedures
- Quality-critical tasks
- Training new agents

**How to Use:**
1. Identify relevant SOP
2. Read entire SOP before starting
3. Follow steps sequentially
4. Complete all checklists
5. Document any deviations
6. Submit for QA review

### For Custom Tasks

**When SOPs Don't Apply:**
- Unique client requirements
- Experimental strategies
- One-time projects
- Innovative solutions

**Process:**
1. Consult brain/workflow-rules.md
2. Document custom approach
3. Get PM or CEO approval
4. Create new SOP if successful
5. Share learnings with team

## SOP Development

### Creating New SOPs

**When to Create:**
- Task performed 3+ times
- Need for consistency identified
- Quality issues discovered
- New service offering

1. Document current process
2. Identify best practices
3. Write step-by-step instructions

**Update Process:**
1. Identify needed changes
2. Update SOP document
3. Test updated process
4. QA review
5. CEO approval
6. Notify all agents
7. Train on changes
8. Archive old version

## SOP Standards

### Quality Requirements

**Every SOP must:**
- Be clear and unambiguous
- Include all necessary steps
- Provide examples where helpful
- Link to relevant templates
- Define quality standards
- Include troubleshooting guide
- Be reviewed by QA agent
- Be approved by CEO

### Formatting Standards

**Structure:**
- Consistent heading hierarchy
- Numbered steps for procedures
- Bullet points for lists
- Tables for comparisons
- Checklists for verification

**Writing Style:**
- Clear, concise language
- Action-oriented verbs
- Second person ("you")
- Active voice
- No jargon (or defined)

## Best Practices

### For SOP Users

1. **Read First:** Always read SOP before starting task
2. **Follow Exactly:** Don't skip steps
3. **Document Deviations:** Note any changes from SOP
4. **Provide Feedback:** Suggest improvements
5. **Keep Updated:** Use latest version

### For SOP Creators

1. **Be Clear:** Write unambiguous instructions
2. **Be Complete:** Include all necessary steps
3. **Be Practical:** Test in real scenarios
4. **Be Visual:** Use diagrams and examples
5. **Be Current:** Update regularly
