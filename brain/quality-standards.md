## Minimum Quality Requirements

### 1. Accuracy & Correctness

**Data Accuracy:**
- All statistics and metrics must be verified
- Cite sources for all data points
- No outdated information (check publication dates)
- Cross-reference multiple sources when possible
- Flag any data that cannot be verified

**Technical Accuracy:**
- All code snippets must be tested and functional
- Technical recommendations must follow current best practices
- No deprecated techniques or tools
- Verify all tool recommendations are current
- Test all examples before including them

**Grammar & Spelling:**
- Zero grammatical errors
- Zero spelling mistakes
- Proper punctuation throughout
- Consistent capitalization
- Professional language (no slang, emojis, or casual tone)

### 2. Completeness

**Required Elements:**
- Clear executive summary (for reports)
- Detailed findings with evidence
- Actionable recommendations
- Implementation guidance
- Timeline estimates
- Expected outcomes
- Next steps

**Completeness Checklist:**
- [ ] All sections of template completed
- [ ] No placeholder text remaining
- [ ] All data points explained
- [ ] All recommendations justified
- [ ] All questions answered
- [ ] All requirements met

### 3. Clarity & Readability

**Writing Standards:**
- Clear, concise language
- Avoid jargon (or define it when necessary)
- Logical flow and structure
- Short paragraphs (3-4 sentences max)
- Bullet points for lists
- Headings to organize content
- White space for readability

**Formatting:**
- Consistent heading hierarchy
- Proper use of bold, italic, underline
- Tables for data comparison
- Charts/graphs where appropriate
- Professional fonts and spacing
- Page numbers in documents

**Accessibility:**
- Alt text for all images
- Descriptive link text (no "click here")
- Sufficient color contrast
- Readable font sizes (12pt minimum)
- Logical reading order

### 4. Professional Presentation

**Visual Design:**
- Clean, professional layout
- Consistent branding (colors, fonts, logos)
- High-quality images and graphics
- Proper alignment and spacing
- No clutter or unnecessary elements
- Professional cover pages

**Document Standards:**
- PDF format for final reports
- PowerPoint for presentations
- Excel for data analysis
- Word for editable documents
- Markdown as source format

**Branding:**
- Agency logo on all documents
- Consistent color scheme
- Professional typography
- Contact information included
- Confidentiality notices when appropriate

### 5. Actionability

**Recommendations:**
- Specific, not vague
- Prioritized by impact and effort
- Include implementation steps
- Provide timelines
- Estimate resources needed
- Define success metrics

**Avoid:**
- Generic advice
- Unclear next steps
- Recommendations without justification
- Overwhelming lists without prioritization
- Suggestions without implementation guidance

### 6. Data Visualization

**Charts & Graphs:**
- Choose appropriate chart type
- Clear labels and titles
- Legend when needed
- Source data included
- Easy to read and understand
- Consistent color scheme

**Tables:**
- Clear column headers
- Proper alignment
- Consistent formatting
- Totals and percentages where relevant
- Source data available
- Easy to scan

**Metrics:**
- Current vs. previous period
- Industry benchmarks when available
- Trend indicators (up/down arrows)
- Color coding for performance
- Context for all numbers

## Review & Approval Process

### Three-Tier Review System

**Tier 1: Self-Review (Agent)**
- Agent reviews own work before submission
- Checks against quality checklist
- Verifies all data and claims
- Ensures completeness
- Proofreads for errors

**Tier 2: QA Agent Review**
- QA agent reviews all deliverables
- Checks against quality-standards.md
- Verifies accuracy and completeness
- Tests any code or technical elements
- Provides feedback and approval

**Tier 3: Final Approval**
- CEO or PM reviews strategic deliverables
- Final sign-off before client delivery
- Ensures alignment with client goals
- Confirms quality standards met

### Review Checklist

**Before Submitting to QA:**
- [ ] All sections completed
- [ ] No placeholder text
- [ ] Data verified and sourced
- [ ] Grammar and spelling checked
- [ ] Formatting consistent
- [ ] Branding applied
- [ ] Action items clear
- [ ] Recommendations justified
- [ ] Timeline realistic
- [ ] Success metrics defined

**QA Review Criteria:**
- [ ] Meets all minimum quality requirements
- [ ] Follows template structure
- [ ] Data accurate and sourced
- [ ] Recommendations actionable
- [ ] No grammatical errors
- [ ] Professional presentation
- [ ] Client-ready quality
- [ ] Aligns with brain/ standards

**Final Approval Criteria:**
- [ ] QA approved
- [ ] Client requirements met
- [ ] Strategic alignment confirmed
- [ ] Quality standards exceeded
- [ ] Ready for client delivery

## Common Quality Issues & Solutions

### Issue 1: Grammatical Errors
**Prevention:**
- Use grammar checking tools (Grammarly, etc.)
- Read aloud to catch errors
- Peer review before QA
- Take breaks before proofreading

**Solution:**
- Fix immediately
- Update templates if pattern emerges
- Provide agent training if needed

### Issue 2: Incomplete Data
**Prevention:**
- Use data collection checklists
- Verify data sources before starting
- Set data gathering milestones
- Flag missing data early

**Solution:**
- Contact client for missing data
- Use industry benchmarks as fallback
- Clearly note data limitations
- Provide alternative analysis if possible

### Issue 3: Vague Recommendations
**Prevention:**
- Use recommendation templates
- Require "why" and "how" for each recommendation
- Set specificity standards
- Review against actionability criteria

**Solution:**
- Rewrite with specific steps
- Add implementation guidance
- Include timeline and resources
- Define success metrics

### Issue 4: Poor Formatting
**Prevention:**
- Use standardized templates
- Apply formatting guidelines
- Review visual presentation
- Use style guides

**Solution:**
- Reformat to standards
- Update template if needed
- Provide formatting training
- Implement formatting checklist

### Issue 5: Outdated Information
**Prevention:**
- Check publication dates
- Verify current best practices
- Use recent sources (< 2 years old)
- Set information freshness standards

**Solution:**
- Update with current information
- Note when information was current
- Provide alternative if outdated
- Update knowledge-base/ if needed

## Exception Handling

2. Explain why standard cannot be met
3. Propose alternative solution
4. Get approval from QA agent
5. Document exception with justification
6. Implement solution

**Approval Required:**
- QA agent for minor exceptions
- CEO agent for major exceptions
- Client for client-facing exceptions

**Documentation:**
- Record all exceptions
- Track exception patterns
- Use exceptions to improve standards
- Share learnings with team
