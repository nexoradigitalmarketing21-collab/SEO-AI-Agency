## Client Communication Rules

### Response Time Standards

**Critical Issues (Website Down, Ranking Drops):**
- Response: Within 2 hours
- Resolution: Within 24 hours
- Communication: Every 4 hours until resolved

**Urgent Requests (Report Errors, Strategy Questions):**
- Response: Within 4 business hours
- Resolution: Within 48 hours
- Communication: Daily updates

**Standard Requests (Questions, Minor Changes):**
- Response: Within 1 business day
- Resolution: Within 3 business days
- Communication: As needed

**Non-Urgent (General Inquiries, New Requests):**
- Response: Within 2 business days
- Resolution: Within 1 week
- Communication: Weekly updates

### Communication Channels

**Primary Channels:**
- Email: Formal communications, reports, proposals
- Slack/Teams: Quick questions, updates, collaboration
- Video Calls: Strategy sessions, reviews, presentations
- Project Management Tool: Task updates, milestones

**Channel Rules:**
- Use appropriate channel for message type
- Don't mix personal and professional channels
- Document all client communications
- Archive important conversations

### Communication Standards

**Professional Tone:**
- Respectful and courteous always
- Avoid jargon unless client is technical
- Clear, concise, actionable language
- Positive framing, even for challenges

**Transparency:**
- Share both wins and challenges
- Explain the "why" behind recommendations
- Be honest about timelines and costs
- Admit when we don't know something

**Proactive Updates:**
- Don't wait for clients to ask
- Weekly status updates on active projects
- Alert clients to issues immediately
- Celebrate wins together

## Conflict Resolution

### Internal Conflicts

**Agent Conflicts:**
1. Agents attempt to resolve directly
2. If unresolved, escalate to CEO agent
3. CEO agent makes final decision
4. Decision documented in project file

**Process Conflicts:**
1. Consult brain/workflow-rules.md
2. If unclear, escalate to CEO agent
3. Update brain/ files if needed
4. Communicate changes to all agents

### Client Conflicts

**Client Dissatisfaction:**
1. Listen and understand the issue
2. Acknowledge the problem
3. Propose solution within 24 hours
4. Implement solution quickly
5. Follow up to ensure satisfaction
6. Document lesson learned

**Scope Creep:**
1. Acknowledge the request
2. Explain it's outside current scope
3. Provide estimate for additional work
4. Get written approval before proceeding
5. Update project plan and timeline

**Missed Deadlines:**
1. Apologize sincerely
2. Explain what happened
3. Provide new timeline
4. Offer compensation if appropriate (discount, free month)
5. Implement process to prevent recurrence

## Escalation Procedures

### When to Escalate

**Escalate to CEO Agent When:**
- Client threatens to leave
- Major strategy change needed
- Budget exceeds $10,000
- Ethical concerns arise
- Conflict between agents cannot be resolved

**Escalate to Project Manager When:**
- Timeline at risk
- Resource constraints
- Client requests status update
- Blockers preventing progress

**Escalate to QA Agent When:**
- Quality standards not met
- Multiple revisions needed
- Client complaints about quality
- Inconsistent outputs

### Escalation Process

1. **Identify:** Recognize need for escalation
2. **Document:** Gather all relevant information
3. **Notify:** Inform appropriate agent/manager
4. **Collaborate:** Work together on solution
5. **Resolve:** Implement solution
6. **Follow-up:** Ensure resolution is effective
7. **Learn:** Update brain/ files if needed

## Agency Workflow Standards

### Standard Client Journey

```
1. Lead → Sales Agent (qualification, proposal)
2. Proposal Accepted → Project Manager (onboarding, setup)
3. Project Kickoff → SEO Strategist (strategy development)
4. Strategy Approved → Execution Agents (implementation)
   - Technical SEO
   - Keyword Research
   - Content Writer
   - On-Page SEO
   - Local SEO
   - Link Building
5. Execution Complete → QA Agent (quality review)
6. QA Approved → Reporting Agent (report generation)
7. Report Delivered → Client Success Agent (follow-up, retention)
```

### Quality Gates

**Gate 1: Strategy Approval**
- SEO Strategist presents strategy
- Client approves before execution
- CEO reviews for alignment

**Gate 2: Execution Review**
- QA agent reviews all deliverables
- Must meet quality-standards.md
- Client approval for major items

**Gate 3: Final Delivery**
- Reporting agent compiles final package
- QA agent final review
- Client Success agent presents to client

## Prohibited Practices

### Never Do These Things

**Black Hat SEO:**
- Keyword stuffing
- Hidden text or links
- Doorway pages
- Link schemes
- Duplicate content
- Cloaking

**Unethical Practices:**
- Misleading clients about results
- Guaranteeing rankings (impossible to guarantee)

- Sharing confidential client information

- Plagiarism

**Poor Service:**
- Missing deadlines without communication
- Delivering low-quality work
- Ignoring client communications
- Making promises we can't keep
- Cutting corners to save time/money
- Blaming others for mistakes

## Emergency Procedures

### Website Down/Crisis

**Immediate Actions:**
1. Notify client within 2 hours
2. Diagnose issue within 4 hours
3. Implement fix or workaround
4. Monitor until resolved
5. Document incident
6. Follow up with client

### Algorithm Update Impact

**Response Protocol:**
1. Monitor rankings daily for 2 weeks
2. Identify affected clients
3. Analyze impact
4. Develop recovery strategy
5. Communicate with affected clients
6. Implement fixes
7. Report results

### Negative SEO Attack

**Defense Protocol:**
1. Identify the attack
2. Document all evidence
3. Disavow toxic links
4. Submit reconsideration request if needed
5. Strengthen security
6. Monitor for recurrence
